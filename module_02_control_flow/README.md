# Module 02: Control Flow

За замовчуванням програма виконується рядок за рядком зверху вниз. **Керування потоком виконання (control flow)** дозволяє «ставити розвилки»: виконувати одні гілки коду і пропускати інші залежно від умов. У цьому модулі головний інструмент — умовні оператори `if / elif / else` і логічні оператори `and / or / not`.

## Ідея умов: програма приймає рішення

**Умовний оператор** дозволяє виконувати блок коду тільки тоді, коли деякий вираз істинний.

```python
age = 20

if age >= 18:
    print("Adult")
```

Тут код у блоці `if` виконається лише якщо умова `age >= 18` істинна. Якщо ні — програма просто піде далі.  

Типові місця, де потрібні умови:

- перевірка прав доступу (логін/пароль, роль користувача);  
- реакція на введені значення (негативне/позитивне число, коректний діапазон);  
- вибір режиму роботи (який пункт меню обрано).  

***

## if / elif / else: «драбинка» взаємовиключних випадків

### if + else

Проста розвилка «так / ні»:

```python
number = 5

if number % 2 == 0:
    print("Even")
else:
    print("Odd")
```

- якщо умова в `if` істинна — виконується блок під `if`;  
- якщо хибна — виконується блок `else`.

### if / elif / else

Коли варіантів більше ніж два, використовують «драбинку»:

```python
score = 75

if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
elif score >= 60:
    grade = "D"
else:
    grade = "F"
```

Особливості такої структури: 

- умови перевіряються **згори вниз**;  
- як тільки одна умова істинна — її блок виконується, решта пропускаються;  
- `else` спрацьовує лише якщо **жодна** з попередніх умов не спрацювала;  
- `else` завжди йде внизу.  

Це зручно для **взаємовиключних** випадків: система оцінювання, діапазони значень, «меню» з кількома пунктами.

## Логічні оператори and / or / not

Часто умова — це не одне порівняння, а комбінація кількох. Тут вступають у гру логічні оператори.

- `and` — істинно, якщо **обидві** умови істинні.  
- `or` — істинно, якщо істинна **хоча б одна** умова.  
- `not` — заперечує результат (True → False, False → True).  

Приклади:

```python
age = 25
has_ticket = True

if age >= 18 and has_ticket:
    print("You can enter")

if age < 18 or not has_ticket:
    print("You cannot enter")
```

Комбінації дозволяють компактно записувати складні перевірки:  
«Користувач старше 18 і має квиток», «або молодший 18, або без квитка» тощо.

Важливо пам’ятати, що в логічних виразах теж є порядок обчислення: спочатку `not`, потім `and`, потім `or`.

## Вкладені умови та читабельність

Умови можна **вкладати** одна в одну:

```python
if is_logged_in:
    if is_admin:
        print("Welcome, admin")
    else:
        print("Welcome, user")
else:
    print("Please log in")
```

Але глибоке вкладення швидко робить код важким для читання. Часто краще використовувати комбіновані умови (`if is_logged_in and is_admin:`) або розбивати логіку на функції.

***

## Control flow як фундамент

Умовні оператори — це перший крок до того, щоб програма:

- **реагувала** на різні ситуації (різні дані → різні дії);  
- **обробляла помилкові випадки** (некоректні значення, заборонений доступ);  
- будувала логіку «якщо…, інакше…».  

Далі ті самі ідеї будуть використовуватися в поєднанні з циклами, обробкою помилок та патернами рівня «state machine», але все починається з простого `if`.
## Огляд вправ модуля

У цьому модулі 6 практичних задач, які покривають різні типи умов та комбінації операторів.

### Task 1: Even or odd

- Ввід: одне ціле число.
- Вивід: `"Even"` для парних чисел, `"Odd"` для непарних.
- Цілі: використання `%` (остача від ділення), простий `if/else`.


### Task 2: Positive, negative or zero

- Ввід: одне число (може бути з комою).
- Вивід: `"Positive"`, `"Negative"` або `"Zero"`.
- Цілі: ланцюжок `if / elif / else`, правильний порядок умов.


### Task 3: Grading system

- Ввід: оцінка від 0 до 100.
- Вивід: літера `A`, `B`, `C`, `D` або `F` залежно від діапазону.
- Цілі: побудова «драбинки» умов, перевірка діапазонів зверху вниз.

Подібні приклади використовують у навчальних статтях про умовні оператори з системою оцінювання.

### Task 4: Simple login system

- Ввід: логін та пароль.
- Вивід: `"Login successful"` при правильних даних, `"Invalid credentials"` – при помилці.
- Цілі: логічний `and`, порівняння рядків, простий сценарій авторизації.


### Task 5: Max of three numbers

- Ввід: три числа.
- Вивід: найбільше з них.
- Цілі: порівняння кількох значень, покрокове оновлення змінної `max_value`, практика вкладених або послідовних `if`.


### Task 6 (optional): Simple calculator with operator

- Ввід: два числа та символ операції (`+`, `-`, `*`, `/`).
- Вивід: результат обчислення або повідомлення про помилку для невідомої операції.
- Додатково: обробка ділення на нуль.
- Цілі: розгалуження за значенням змінної (`if/elif/else`), робота із символами, обробка помилкових випадків.
