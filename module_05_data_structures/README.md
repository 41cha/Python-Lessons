# Module 05: Data Structures

У Python є кілька базових структур даних для зберігання колекцій: **список**, **кортеж**, **множина** і **словник**. Вони схожі тим, що дозволяють тримати багато значень в одній змінній, але ведуть себе по‑різному і підходять для різних задач. Уміння **обирати правильну структуру** — одна з ключових навичок Python‑розробника. 

## Загальна ідея: коли що використовувати

Якщо сильно спростити: 

- **List (список)** — впорядкована змінна колекція.  
  Використовуйте, коли:
  - порядок елементів важливий;  
  - потрібно часто додавати/видаляти/міняти елементи;  
  - допускаються дублікати.  

- **Tuple (кортеж)** — впорядкована **незмінна** колекція.  
  Використовуйте, коли:
  - дані логічно пов’язані й мають фіксовану структуру (наприклад, координати `(x, y)`);  
  - ці дані не мають змінюватися після створення;  
  - зручно повертати кілька значень з функції. 

- **Set (множина)** — неупорядкована колекція **унікальних** елементів.  
  Використовуйте, коли:
  - важлива **унікальність** (прибрати дублікати);  
  - потрібні швидкі перевірки «чи є елемент у наборі»;  
  - потрібні операції над множинами: перетин, об’єднання, різниця.

- **Dict (словник)** — колекція пар `ключ → значення`.  
  Використовуйте, коли:
  - потрібно швидко знаходити значення за ключем (як у справжньому словнику — слово → значення);  
  - є структуровані дані: профілі користувачів, налаштування, мапа «слово → кількість».

***

## List: універсальна змінна послідовність

Список — це впорядкована колекція в `[]`, яку можна змінювати:

```python
numbers = [10, 20, 30]
numbers.append(40)   # додати в кінець
numbers[0] = 25      # змінити елемент
```

Важливі властивості:

- **порядок зберігається**;  
- можна звернутися за індексом: `numbers[0]`, `numbers[-1]`;  
- можна робити зрізи: `numbers[1:3]`;  
- можна додавати, видаляти, сортувати.  

Типові сценарії:

- списки користувачів, замовлень, повідомлень;  
- результати запиту (список записів);  
- часові ряди, колекції значень, які ростуть.

***

## Tuple: фіксований набір значень

Кортеж схожий на список, але **його не можна змінити** після створення:

```python
point = (3, 5)
x, y = point
```

Коли це корисно:

- ви хочете гарантувати, що структура «пара значень» не зміниться випадково;  
- ви повертаєте з функції кілька значень (`return x, y`) і зручно розпаковувати;  
- кортеж може використовуватися як ключ у словнику (на відміну від списку), бо він незмінний.

Приклад: координати, дата `(year, month, day)`, точка налаштувань, де значення не редагуються.

***

## Set: унікальні значення і операції над множинами

Множина — це «мішок» унікальних елементів без гарантованого порядку: 

```python
nums = [1, 2, 2, 3, 4, 4]
unique = set(nums)   # {1, 2, 3, 4}
```

Сильні сторони:

- автоматично прибирає дублікати;  
- дуже швидко перевіряє, чи є елемент у множині: `if x in my_set`;  
- підтримує математичні операції:
  - `a & b` — перетин;  
  - `a | b` — об’єднання;  
  - `a - b` — різниця.

Приклади використання:

- пошук унікальних значень (унікальні email, теги);  
- визначення спільних елементів між наборами (спільні друзі, перетин множин прав);  
- швидкі перевірки «чи входить цей елемент до дозволених».

***

## Dict: ключ → значення

Словник — це структура «ключ → значення» у `{}`:

```python
student = {
    "name": "Andrii",
    "age": 20,
    "city": "Vinnytsia",
}

print(student["name"])   # доступ за ключем
student["age"] = 21      # оновити значення
student["grade"] = 95    # додати нове поле
```

Ключові моменти:

- ключі мають бути **унікальними**;  
- доступ за ключем дуже швидкий (всередині — хеш‑таблиця);
- зручно моделювати «об’єкти» без класів (профілі, конфіги).  

Частий патерн — **словник лічильників**: слово → скільки разів зустрічається в тексті.

***

## Як ці структури працюють разом

У реальних задачах структури комбінуються:

- список словників: `list[dict]` — список студентів, де кожен студент — словник полів;  
- список кортежів: `list[tuple]` — список точок, координат або пар значень;  
- множина з елементів списку — щоб прибрати дублікати;  
- словник, де ключ — кортеж, а значення — список тощо.

Приклад:

```python
students = [
    {"name": "Andrii", "age": 20},
    {"name": "Maria", "age": 19},
    {"name": "Oleh", "age": 22},
]

names = [s["name"] for s in students]  # список імен
ages = [s["age"] for s in students]    # список віків
avg_age = sum(ages) / len(ages)
```

Це типовий шаблон для роботи з табличними/JSON‑подібними даними.

***

## Як думати при виборі структури

Добре запитати себе: 

1. **Чи важливий порядок?**  
   - Так, і елементи можуть змінюватися — **list**.  
   - Так, але дані фіксовані — **tuple**.  

2. **Чи потрібна унікальність?**  
   - Так — **set**.  

3. **Чи є природне «ім’я» для кожного елемента?**  
   - Так, мені потрібні пари ключ → значення — **dict**.  

4. **Чи ймовірно, що структура буде рости / змінюватися?**  
   - Так — зазвичай **list** або **dict**.  
   - Ні, це фіксований набір — **tuple**.  

Цей модуль не тільки показує синтаксис структур даних, а й формує інтуїцію: **яку «форму» дати своїм даним, щоб з ними було зручно працювати й читати код через місяць**.

### Task 4: Tuple for coordinates

### Task 5: Dictionary of student

- Створення словника `student` з полями `name`, `age`, `city`.  
- Вивід значень, оновлення міста та додавання нового поля `grade`.  
- Тренуються: доступ до значень `dict[key]`, додавання та зміна пар ключ–значення.

### Task 6: Word frequency (dict)

- Рядок `"python is fun and python is easy"` потрібно розбити на слова та порахувати частоту кожного слова.  
- Результат — словник `{слово: кількість}`.  
- Тренуються: `split()`, цикл по словах, патерн «словник лічильників»: якщо слово вже є — збільшити, інакше створити з 1.

### Task 7: Set operations

- Є множини `a` та `b`, треба знайти:  
  - перетин `a & b`;  
  - об’єднання `a | b`;  
  - різницю `a - b`.  
- Тренуються: базові операції над множинами для пошуку спільних, усіх та унікальних елементів.

### Task 8 (optional): List of dictionaries

- Список `students`, де кожен елемент — словник `{"name": ..., "age": ...}`.  
- Потрібно вивести всі імена та знайти середній вік.
